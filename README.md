# Проект: Стенд вычислительных экспериментов A2->T (A2->T Lab)

## 1. Общая задача проекта
Разработка системы для экспериментального исследования алгоритмов на уровне их логической абстрактной сложности. Система состоит из интерпретатора расширенного псевдокода и автоматизированного стенда для пакетного запуска экспериментов. 

Фундаментальное архитектурное требование: сбор надежных метрик осуществляется через детерминированные абстрактные счетчики операций (сравнения, присваивания, обращения к памяти), а не через системные таймеры времени выполнения. Это обеспечивает полную воспроизводимость результатов эксперимента независимо от аппаратной среды.

## 2. Текущий фокус разработки: Бэкенд пакетного стенда (Batch Runner)
В данный момент разработка сфокусирована на реализации изолированной среды выполнения (headless-режим) , генерации входных данных и статистической редукции собранных метрик.

Текущая подзадача разбита на два независимых функциональных блока.

### Блок А: Генерация данных и изоляция (Data Generation & Isolation)
Стенд должен автоматизировать запуск тысяч прогонов (trials) при заданных параметрах (например, размере массива $N$).

***Библиотека генераторов (Generators):** Необходима реализация функций, создающих входные структуры данных для алгоритмов.
  * Случайные массивы и др структуры данных
  * Строки
  * Графы
  * и т.д.

***Механизм изоляции (State Reset):** Каждый отдельный прогон алгоритма должен выполняться в абсолютно "чистой" среде. Перед каждым запуском (trial) стенд обязан сбрасывать:
  1. Память переменных и структуры данных.
  2. Значения всех счетчиков операций (comparisons, swaps, total_ops и т.д.).

### Блок Б: Агрегация метрик (Metric Aggregation)
После выполнения серии прогонов для конкретного набора параметров стенд собирает "сырые" данные (векторы счетчиков для каждого trial). Эти данные необходимо статистически обработать.

* **Входные данные для агрегатора:** Массив объектов/словарей, содержащих результаты каждого запуска (значение $N$, номер прогона, итоговые значения счетчиков).
* **Вычисляемые статистические оценки:**
  * Среднее арифметическое (Mean).
  * Медиана (Median) — как робастная метрика, устойчивая к выбросам.
  * Квантили (0.25 и 0.75) для оценки распределения.
  * Дисперсия / Стандартное отклонение (Variance/STD) для оценки стабильности вероятностных алгоритмов.

## 3. Требования к коду
* **Слабая связность:** Модуль генерации и модуль статистики не должны напрямую зависеть от реализации AST-дерева интерпретатора. Они должны работать с абстрактными интерфейсами `AlgorithmRunner` или `ExperimentContext`.
* **Воспроизводимость:** Модуль генерации данных должен принимать фиксированный `seed`, чтобы гарантировать идентичность сгенерированных массивов при повторном запуске эксперимента.
